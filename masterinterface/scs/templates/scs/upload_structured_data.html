Upload to VPH-Share structured data, like tabular ones, to make use of many time and cost saving facilities:

  • Use data with established tools and applications to personalise workflows.
  • Store large files in our free, fast and easy to use cloud environment.
  • Share data with researchers, clinicians and developers to maximise its potential.

In VPH-Share we primarily refer to tabular, and potentially relational, data sources such as CSV extracts and relational databases, as Structured Data.


A desktop application, Data Publication Suite (DPS), has been developed to support the process of publishing clinical or research data sets in a secure internet accessible way. The data once published may browsed using an RDF Browser or queried using SPARQL and an SQL type protocol from the OGSA-DAI project.


The general process for publication, although many of them are not mandatory, is as follows:

  • Import a data source
  • Define relationships between the tables if they exist and are not automatically detected
  • Semantically annotate the data
  • Create a destination container of the server (if you have permissions)
  • Create a new destination based on a data source
  • Define a de-identification profile for this destination
  • Publish the data
  • Manage the access list for the resource
